A static site does not have to mean a frozen brochure. It can have a typed
content pipeline, responsive images, RSS, search, and a few carefully chosen
interactive controls while still shipping predictable files.

The important boundary is intent. Pages and data should resolve during the
build. Browser code should appear only where a reader needs state, such as a
theme toggle, a search field, or a recipe scaler.

That boundary is what this Nib example is testing against the personal site.
